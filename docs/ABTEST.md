# A/B Tests

## Test 1: Individual Task Card Layout

User Story Number: US4 Golden Path

Metrics: From the HEART framework, adjusting the task card will focus on the retention and task success metrics.

Hypothesis: Adjusting the layout of individual task cards will improve user engagement and task completion rates. Enhanced visual elements may make it easier for users to quickly identify and interact with their tasks, leading to increased retention and higher task success rates.

Experiment: We will create two variations of the task card layout so that we have the current design and a new design that incorporates improved visual cues, such as different ways to show task priority as well as what date the task needs to be completed by. We will randomly assign users to either the control or variation group and track their interactions with the task cards over a period time. This will be a 50/50 split of the user base. We will measure retention by tracking how often users return to the app and task success by monitoring the completion rates of tasks in both groups.

Variations: More than one new layout could be created for the task cards. For example, one variation could focus on a more minimalist design with less information displayed, while another variation could include additional features such as color-coding for task priority or due dates. This would allow us to compare multiple design changes and determine which elements have the most significant impact on user engagement and task completion rates. This could be tested alongside the original layout such that there are three or even four groups of users to compare against each other.

## Test 3: Profile Customization Options 

User Story Number: US3

Metrics: Happiness, Engagement

Hypothesis: Many users like having the ability to create custom profiles in apps to try and distinguish themselves from other users, but sometimes too much personalization can feel overwhelming, or even unnecessary based on the type of app. If there are too many options in setting up a profile, this can lead to a lower number of users actively using the app due to the profile being too complicated to set up. Conversely, if there are too litte or not enough options for personalization, this can lead to user's not being as happy with utilizing the app, and can result in negative reviews. We can hypothesize that buy limiting the number of personalization options, some users will experience a decrease in happiness when using the app due to the lack of personalization that comes with creating their profile in our app. 

Experiment: Using Firebase Analytics, we can test this hypothesis based on user reviews of the app, as well as the active engagement time and user retention metrics to view how often the users would return based on the different variations between the A test and the B test. Because our auidence is primarily college aged students, the userbase is fairly massive, which can result in more accurate results. Because of this, we want to be able to make this experiment available to as many users as possible to get the best possible feedback on the app.
